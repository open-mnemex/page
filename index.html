I — Structure

The Architecture

Five domains. One system of truth.

~/Documents ├── 00_Vault/ # Immutable facts ├── 10_Projects/ # Active work with deadlines ├── 20_Areas/ # Capabilities you maintain ├── 30_Resources/ # External collected materials ├── 99_Archives/ # Completed projects └── __To_Review/ # Safe staging area
II — Philosophy

Three Principles

The thinking behind the structure.

Fig. 1 — File Lifecycle
01

Fact vs Fiction

Facts are immutable. A tax return, once filed, never changes. Creations are volatile. Drafts evolve until they ship.

02

Capability vs Collection

Areas are skills you possess and systems you actively maintain. Resources are external materials you've gathered.

03

Flow

Files enter via Downloads. Active work happens in Projects. Final artifacts move to Vault. Dead projects move to Archives.

04

No Deletion

Files are never deleted. They are moved to Trash, archived, or frozen. Every artifact has a history. Nothing is lost.

III — The Pattern

CLAUDE.md / README.md

The core design pattern of mnemex.

Fig. 2 — Dual File Pattern

Shared rules. Private data.

CLAUDE.md files define the framework — naming conventions, protocols, filing rules. They're tracked in git and shared with everyone who clones the repo.

README.md files hold your personal context — entity names, project inventories, filing logs. They're gitignored, keeping your data private.

Clone the repo. Copy each README.example.md to README.md. Fill in your life.

IV — Skills

Voice commands for your digital archivist.

Document Management

Route files from ~/Downloads into the system with VNS naming conventions. Auto-detects file type and suggests the correct domain.
/capture
Quick-capture quotes, books, articles, tools, letters, and any content worth keeping into Resources.
/pdf
Full PDF toolkit: extract text and tables, create new PDFs, merge, split, and fill forms programmatically.

Generate and edit images using Gemini's multimodal model. Supports up to 14 input images for composition.
Project Lifecycle
/project init
Create a new project with the standard skeleton, control plan, and folder structure.
/project archive
Archive a completed project using the 3R Protocol: Refine, Reduce, Record.
/project reorg
Reorganize a project's internal structure when scope changes or files accumulate.
/project link
Create symlinks between Vault evidence files and active project folders.

V — Begin

Getting Started

Four steps to an organized life.

Install Claude Code, Anthropic's CLI for working with Claude.
Clone the framework:
git clone https://github.com/open-mnemex/core.git ~/Documents
Copy each README.example.md to README.md in the same folder. Fill in your personal data: entity names, project lists, area inventories.
Run /process-inbox to start filing documents from your Downloads folder.
